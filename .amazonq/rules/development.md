# Regras Gerais de Desenvolvimento

## Qualidade de Código

- Prefira código legível a código "inteligente"
- Use nomes descritivos para variáveis e funções
- Evite comentários desnecessários - código deve ser autoexplicativo
- Mantenha funções pequenas (máximo 20 linhas quando possível)

## Git e Versionamento

- Commits pequenos e frequentes
- Mensagens de commit descritivas
- Use conventional commits quando possível

## Tratamento de Erros

- Sempre trate erros de forma adequada
- Use try/catch para operações assíncronas
- Forneça feedback visual para o usuário

## Acessibilidade

- Use accessibilityLabel em componentes interativos
- Mantenha contraste adequado
- Teste com leitor de tela
